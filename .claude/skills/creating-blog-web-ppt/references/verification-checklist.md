# 验证清单

没有新的验证证据之前，不要声称完成。

## 执行顺序

必须按这个顺序验证：

1. 先做静态检查
2. 再做浏览器检查
3. 最后做收尾清理

## 静态检查

1. 读取最终 HTML，确认：
   - 存在 `<title>`
   - 存在多个 `section.slide`
   - 图片路径相对文章目录正确
   - favicon 路径与 deck 所在位置匹配
2. 对最终 HTML 运行 `ReadLints`。

## 浏览器检查

从仓库根目录启动一个临时静态服务，再用浏览器打开最终页面。

至少确认：

- 页面能正常打开
- 页面标题正确
- slide 数量大于 0
- 初始页码正确
- 执行一次翻页操作后页码发生变化
- 浏览器控制台没有报错

## 最低证据格式

验证证据至少要覆盖：

- 打开的 URL
- 页面标题
- slide 数量
- 翻页前页码
- 翻页后页码
- 控制台错误数量

建议按下面这个格式回报：

```markdown
验证结果：
- 打开的 URL：`...`
- 页面标题：`...`
- slide 数量：`...`
- 翻页前页码：`...`
- 翻页后页码：`...`
- 控制台错误：`0`
```

## 验证失败时怎么回报

如果验证没有通过，不要说“已完成”。

至少要明确回报：

- 失败发生在哪一步
- 实际看到的现象
- 是否已经定位到原因
- 下一步准备怎么修

建议按下面这个格式回报：

```markdown
当前还不能算完成。

验证失败点：
- 步骤：`浏览器检查`
- 现象：`翻页后页码没有变化`
- 原因判断：`疑似页码更新逻辑未绑定当前索引`
- 下一步：`修复脚本后重新打开页面验证`
```

## 收尾清理

验证完成后：

- 关闭临时静态服务
- 删除本次验证产生的临时浏览器产物

## 这些情况不能算完成

- 浏览器没有打开最终的同目录 HTML
- deck 依赖一个已经删除的中间文件
- 控制台还有报错
- 本地静态服务下图片路径失效
- 翻页后页码没有变化
